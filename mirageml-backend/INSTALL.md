# 🚀 Инструкция по установке и запуску MirageML с PostgreSQL

## 📋 Требования

Перед началом установки убедитесь, что у вас установлены:

- **Node.js** версии 16 или выше
- **PostgreSQL** версии 12 или выше
- **npm** или **yarn**

---

## 🔧 Шаг 1: Установка PostgreSQL

### Windows

1. Скачайте PostgreSQL с официального сайта: https://www.postgresql.org/download/windows/
2. Запустите установщик и следуйте инструкциям
3. Запомните пароль пользователя `postgres`
4. Порт по умолчанию: `5432`

### Проверка установки

```bash
psql --version
```

---

## 📦 Шаг 2: Установка зависимостей проекта

```bash
cd D:\Git\mirageml\mirageml-backend
npm install
```

---

## ⚙️ Шаг 3: Настройка конфигурации

### 3.1. Создайте файл `.env`

Файл `.env` уже создан в папке `mirageml-backend`. Отредактируйте его при необходимости:

```env
# PostgreSQL Database Configuration
DB_HOST=localhost
DB_PORT=5432
DB_NAME=mirageml
DB_USER=postgres
DB_PASSWORD=ваш_пароль_postgres

# Server Configuration
PORT=3001
JWT_SECRET=8ddfda05949bcc8057da59d2b7e62b4f3e12f00d6af892704d87530ae6731cab
```

> **Важно:** Замените `ваш_пароль_postgres` на реальный пароль от PostgreSQL!

---

## 🗄️ Шаг 4: Инициализация базы данных

### Автоматическая инициализация (рекомендуется)

```bash
npm run db:init
```

Эта команда:
1. Создаст базу данных `mirageml`
2. Создаст все необходимые таблицы
3. Загрузит начальные данные (пользователи)

### Миграция данных из JSON (если обновляетесь со старой версии)

Если у вас есть JSON файлы с данными, выполните миграцию:

```bash
npm run db:migrate-data
```

После успешной миграции JSON файлы можно удалить.

### Ручная инициализация (альтернатива)

Если автоматическая инициализация не работает, выполните SQL-скрипты вручную:

```bash
# Подключитесь к PostgreSQL
psql -U postgres

# Создайте базу данных
CREATE DATABASE mirageml;

# Подключитесь к новой базе данных
\c mirageml

# Выполните скрипт создания таблиц
\i D:/Git/mirageml/mirageml-backend/database/001_schema.sql

# Выполните скрипт начальных данных
\i D:/Git/mirageml/mirageml-backend/database/002_initial_data.sql

# Выход
\q
```

## 🛠️ Решение проблем

### Ошибка: "Connection refused"

**Проблема:** PostgreSQL не запущен

**Решение:**
```bash
# Windows - проверьте службу PostgreSQL
# Пуск → Службы → PostgreSQL → Запустить
```

### Ошибка: "password authentication failed"

**Проблема:** Неверный пароль PostgreSQL

**Решение:**
1. Откройте файл `.env`
2. Измените `DB_PASSWORD` на правильный пароль
3. Перезапустите сервер

### Ошибка: "database does not exist"

**Проблема:** База данных не создана

**Решение:**
```bash
npm run db:init
```

### Ошибка: "port 3001 already in use"

**Проблема:** Порт 3001 занят

**Решение:**
1. Откройте `.env`
2. Измените `PORT=3001` на другой порт (например, `PORT=3002`)
3. Перезапустите сервер
